# Study03 - 업로드/인덱싱 파이프라인 학습

## 1) 전체 흐름
1. 사용자가 문서 업로드
2. Gateway가 파일 저장(S3/로컬)
3. 작업 큐 enqueue
4. Worker가 엔진 ingest 호출
5. Qdrant 업서트 완료

## 2) 왜 비동기 큐를 쓰는가
- 대용량 문서 처리 시 API 응답성 확보
- 재시도/백오프/모니터링 용이
- 워커 수평 확장 가능

## 3) 실패 처리 학습
- 파싱 실패(PDF 손상)
- 모델 연결 실패(Ollama 다운)
- 벡터DB 연결 실패(Qdrant 다운)

각 케이스에서 잡 상태 저장과 재처리 정책을 설계해보세요.
