# Study01 - RAG 기초부터 이 저장소까지

## 1) RAG 기본 개념
RAG는 LLM이 외부 문서를 검색한 뒤 답변 생성에 활용하는 방식입니다.
핵심 단계는 다음 4가지입니다.
1. 문서 수집
2. 청크 분할
3. 임베딩 후 벡터 저장
4. 질의 시 유사문서 검색 + 답변 생성

## 2) 이 저장소에서의 RAG
- 벡터 저장소: Qdrant
- 임베딩/LLM: Ollama 또는 OpenAI
- 엔진: LangChain JS / LlamaIndex Py / Flowise

## 3) 학습 포인트
- 같은 질문을 두 엔진으로 비교해 답변 차이 분석
- topK 변화에 따른 정확도/환각 변화 관찰
- 파싱 품질(PDF/DOCX)에 따라 검색 품질이 크게 달라짐
